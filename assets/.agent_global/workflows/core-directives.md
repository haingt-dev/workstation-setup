# Core Directives

**CRITICAL**: Maintain a stable and clean state at all times.

## Principles
1. **No Dirty State**: Do not leave the user's environment in a broken or "dirty" state without explicit permission or during an active debugging session.
2. **Verify Before Completing**: Always verify that your changes work correctly before marking a task as complete.
3. **Minimal Impact**: Make the smallest changes necessary to accomplish the task. Avoid unnecessary modifications.
4. **Reversibility**: When making significant changes, ensure they can be reversed if needed.
