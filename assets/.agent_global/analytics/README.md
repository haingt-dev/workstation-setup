# Token Usage Analytics

**Status**: Basic implementation (manual logging required)
**Purpose**: Track token usage and costs across projects to optimize ROI

## Overview

This system helps you understand:
- Which projects consume most tokens
- Daily/weekly token usage patterns
- Estimated costs per project
- ROI of different agent usage patterns

## Current Limitation

⚠️ **Manual Logging Required**: Claude doesn't expose token stats programmatically yet. You need to manually log usage.

## Usage

### View Today's Usage
```bash
~/.agent_global/analytics/token-tracker.sh today
# Or use alias:
token-today
```

### View Weekly Summary
```bash
token-week
```

### View Project Usage
```bash
token-project Wildtide
```

### View Overall Summary
```bash
token-summary
```

## Manual Logging

After a Claude session, log your usage:

```bash
# Format: timestamp,project,session_id,input_tokens,output_tokens,total_tokens,cost_usd
echo "$(date +%Y-%m-%d),Wildtide,session123,5000,2000,7000,0.045" >> ~/.agent_global/analytics/token-usage.log
```

Or edit the file directly:
```bash
$EDITOR ~/.agent_global/analytics/token-usage.log
```

## Cost Estimation

Script uses approximate Claude 3.5 Sonnet pricing:
- Input: $3 per million tokens
- Output: $15 per million tokens

Actual costs may vary. Check: https://www.anthropic.com/pricing

## Data Format

CSV file: `~/.agent_global/analytics/token-usage.log`

Columns:
1. `timestamp` - YYYY-MM-DD
2. `project` - Project name (e.g., "Wildtide")
3. `session_id` - Unique session ID (optional)
4. `input_tokens` - Tokens sent to Claude
5. `output_tokens` - Tokens generated by Claude
6. `total_tokens` - Sum of input + output
7. `cost_usd` - Estimated cost in USD

## Analysis Tips

### Optimize Token Usage
1. **Check daily usage**: Identify which projects use most tokens
2. **Compare across agents**: Log Kilo/Antigravity usage separately
3. **Measure ROI**: Time saved vs. cost spent
4. **Optimize context**: Smaller Memory Banks = fewer input tokens

```bash

alias token-week='~/.agent_global/analytics/token-tracker.sh week'
alias token-summary='~/.agent_global/analytics/token-tracker.sh summary'
alias token-project='~/.agent_global/analytics/token-tracker.sh project'

```

## Privacy Note

Token logs are stored locally in:
- `~/.agent_global/analytics/token-usage.log`

They contain:
- Project names
- Token counts
- Timestamps
- NO sensitive content from conversations

Safe to backup and analyze.
